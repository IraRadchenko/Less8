list of commands

  • git config --global user.name - встановлює глобально ім'я, яке буде відображатись в полі автора у коммітах, що ви виконуєте
  • git config --system user.name - встановлює системно ім'я, яке буде відображатись в полі автора у коммітах, що ви виконуєте
  • git config user.name - встановлює локально ім'я, яке буде відображатись в полі автора у коммітах, що ви виконуєте
  • git config --global user.email - встановлює електронну адресу, яка буде відображатись в полі автора у коммітах, що ви виконуєте
  • git --version - показує інформацію про версію Git
  • git --v - скорочено від git --version
  • git init - створює новий локальний репозиторій з заданим ім'ям
  • cd ../ - перехід до іншої папки
  • git status - команда, яка використовується для перегляду статусу файлів у вашому локальному репозиторії
  • git add - індексує вказаний файл для послідуючого комміту
  • git add . - індексує всі файли для послідуючого комміту
  • git add -A - індексує всі файли для послідуючого комміту
  • git commit -m 'повідомлення з коротким описом' - фіксує проіндексовані зміни і зберігає їх у історію версій
  • git restore - відміна індексації файлу
  • git rm-cached 'ім'я файлу'- прибирає конкретний файл із контролю версій, але фізично залишає його на своєму місці
  • git remote add origin https://... - команда використовується для додавання або підключення до віддаленого репозиторію
  • git remote -v - команда для перегляду підключених віддалених репозиторіїв
  • git push --set-upstream origin master
  • git push - завантажує всі зміни локальної гілки до віддаленого репозиторію
